## 1. 화면 개요

iPad Home Screen Widgets는 4가지 크기를 지원한다: Small, Medium, Large, 그리고 iPad 전용 **Extra Large**. Extra Large 위젯은 8x4 그리드 슬롯을 차지하며 홈 화면 전체 너비에 걸친다. iOS 26에서는 위젯 배경에 **Liquid Glass** 소재가 적용되며, 인터랙티브 위젯(iOS 17+)과 Live Activity를 포함한 전체 위젯 패밀리를 제공한다.

| 항목 | 값 |
|------|-----|
| **디바이스** | iPad Pro 13" (1210 x 834pt landscape) |
| **플랫폼** | iPadOS 26 |
| **위젯 크기** | Small, Medium, Large, Extra Large |
| **배경 소재** | `.systemMaterial` (반투명 Liquid Glass) |
| **인터랙티브** | iOS 17+ (Button, Toggle 지원) |
| **iOS 26 특징** | Liquid Glass 배경, 개선된 인터랙티브 위젯 |

## 3. 위젯 크기 상세 (4 sizes)

### 3.1 크기 비교표

| 크기 | iPhone (pt) | iPad (pt) | 그리드 슬롯 | WidgetFamily |
|------|-------------|---------|----------|--------------|
| **Small** | 158 x 158 | **170 x 170** | 2 x 2 | `.systemSmall` |
| **Medium** | 338 x 158 | **360 x 170** | 4 x 2 | `.systemMedium` |
| **Large** | 338 x 354 | **360 x 379** | 4 x 4 | `.systemLarge` |
| **Extra Large** | — | **748 x 360** | 8 x 4 | `.systemExtraLarge` |

> iPhone에서는 Extra Large를 지원하지 않음.

### 3.2 공통 위젯 스타일

| 속성 | 값 | 토큰 참조 |
|------|-----|----------|
| Corner Radius | 20pt | `spacing.radius.xxl` |
| 내부 패딩 | 16pt | `spacing.inset.md` |
| 배경 | `.systemMaterial` (반투명) | `materials.backgroundMaterials.regular` |
| Shadow Blur | 40px | — |
| Frost | blur(12px) | `materials.liquidGlass.regular.medium.frostRadius` |

## 4. Size 1: Small (170 x 170pt)

### 4.1 와이어프레임

```
┌──────────────────────┐
│  170 x 170pt, r:20   │
│                      │
│  ┌──────────────┐    │
│  │ 16pt padding  │    │
│  │               │    │
│  │  날씨 예시:    │    │
│  │  ☀️ 24°       │    │
│  │  서울         │    │
│  │               │    │
│  │  최고 28°     │    │
│  │  최저 18°     │    │
│  │               │    │
│  └──────────────┘    │
│                      │
└──────────────────────┘
```

### 4.2 레이아웃 가이드

```
내부 사용 가능 영역: 170 - (16 x 2) = 138 x 138pt

권장 구성:
  - 상단: 제목 또는 아이콘 (22-34pt 높이)
  - 중앙: 핵심 데이터 (Large Title 크기)
  - 하단: 보조 정보 (Caption/Footnote)
```

| 요소 | 폰트 | 권장 크기 | 토큰 참조 |
|------|------|----------|----------|
| 주요 수치 | Large Title (34pt) | 1줄 | `typography.styles.largeTitle` |
| 부제 | Footnote (13pt) | 1-2줄 | `typography.styles.footnote` |
| 아이콘 | SF Symbol | 22-28pt | — |
| 앱 이름 | Caption2 (11pt) | 위젯 외부 (하단 레이블) | `typography.styles.caption2` |

### 4.3 터치 영역

- Small 위젯은 **단일 탭 영역** (전체가 하나의 Link)
- 내부 버튼/토글 불가 → 앱으로 딥링크만 가능

```

  - 좌측 컬럼 (2/3): 리스트 또는 주요 데이터
```

- Medium 위젯부터 **내부 버튼/토글** 지원 (iOS 17+)

- Button: 44pt 최소 터치 영역 유지

## 8. Liquid Glass 위젯 배경 상세

### 8.1 소재 속성

| 속성 | Light Mode | Dark Mode | 토큰 참조 |
|------|-----------|-----------|----------|

| 배경 Opacity | 0.60 | 0.60 | `materials.backgroundMaterials.regular.bgOpacity` |
| Overlay Opacity | 0.25 | 0.25 | `materials.backgroundMaterials.regular.overlayOpacity` |
| Blur Radius | 100px | 100px | `materials.backgroundMaterials.regular.blurRadius` |
| Corner Radius | 20pt | 20pt | `spacing.radius.xxl` |
| Shadow | `#000` a:0.08, blur 24 | `#000` a:0.2, blur 24 | — |

## 9. Light / Dark Mode 차이

| 요소 | Light | Dark |
|------|-------|------|
| 위젯 배경 | 밝은 반투명 (배경화면 비침) | 어두운 반투명 (배경화면 비침) |
| 텍스트 대비 | 어두운 텍스트 on 밝은 배경 | 밝은 텍스트 on 어두운 배경 |
| 그래프/차트 | 진한 색상 | 밝은 색상 (가시성) |
| Shadow | 약한 (a: 0.08) | 강한 (a: 0.2) |
| 위젯 레이블 (외부) | `#000` + shadow | `#fff` + shadow |

## 10. iPad 전용 적응사항

### 10.1 Landscape vs Portrait 크기 차이

| 크기 | Landscape 그리드 | Portrait 그리드 |
|------|-----------------|-----------------|
| Small | 6열 중 2열 차지 | 5열 중 2열 차지 |
| Medium | 6열 중 4열 차지 | 5열 중 4열 차지 |
| Large | 6열 중 4열 차지 | 5열 중 4열 차지 |
| Extra Large | 6열 전체 (+ 여백 활용) | 5열 전체 |

### 10.2 Pointer 지원

| 인터랙션 | 동작 |
|---------|------|
| Hover | 위젯에 미묘한 lift 효과 |
| Click | 딥링크로 앱 열기 |
| Click (인터랙티브 버튼) | 해당 인터랙션 실행 (토글, 버튼 등) |
| Long press | 위젯 편집/삭제 메뉴 |
| Right-click | 위젯 편집/삭제/앱 열기 컨텍스트 메뉴 |

### 10.3 키보드

- 홈 화면에서 키보드 단축키로 직접 위젯 조작 불가
- `⌘Space`로 Spotlight 검색 → 위젯 관련 앱 접근

## 11. 멀티태스킹 고려사항

### 11.1 홈 화면 위젯

- 위젯은 홈 화면에서만 표시 (Split View 앱 내부에는 배치 불가)
- Stage Manager 배경에서 위젯 업데이트 계속 수행 (Timeline Provider)

### 11.2 위젯 업데이트 주기

| 업데이트 유형 | 주기 |
|-------------|------|
| Timeline | `TimelineProvider`에 의해 결정 |
| 최소 주기 | 5분 (시스템 제한) |
| Push 업데이트 | WidgetKit reload trigger |
| Background refresh | Budget 기반 (일 40-70회) |
